## What This Is

A **Classification Tree test design tool** (分類木テスト設計ツール) — a browser-based SPA for creating hierarchical classification trees and mapping leaf nodes to test cases in a matrix view. No backend, no persistence; all state is in-memory React state.

## Commands

```bash
npm run dev       # Start Vite dev server (http://localhost:5173)
npm run build     # Production build → dist/
npm run preview   # Preview production build
npm run lint      # ESLint check
```

No test runner is configured.

## Architecture

The entire application lives in **`src/App.jsx`** — one file, no routing, no component library.

### Data model

```js
{
  trees: [{ id, root: TreeNode }],   // classification trees
  testCases: [{ id, sel: { [leafId]: bool } }]
}
```

`TreeNode` is `{ id, name, children: TreeNode[] }`. The tree structure is immutable-style — every mutation returns a new tree via recursive helpers (`updateNode`, `deleteNodeFrom`, `extractNode`, `insertAfter`).

### Layout

`computeLayout(root, colOffset, treeId)` flattens the tree into positioned node objects with pixel `cx`/`cy` computed from constants (`COL_W=100`, `ROW_H=78`, etc.). Layout is recomputed via `useMemo` on every tree change.

### Key state

| State | Purpose |
|---|---|
| `data` | The entire document (trees + testCases) |
| `editMode` | Toggles edit UI (add/delete buttons, drag, name editing) |
| `editing` | `{ nodeId, treeId, value }` — the node currently being renamed |
| `drag` | `{ nodeId, treeId, ghostName, ghostX, ghostY, targetId, targetTreeId }` |

### Drag-and-drop

Mouse event listeners (`mousemove`/`mouseup`) are attached to `document` only while dragging (via `useEffect` keyed on `!!drag`). Stale-closure issues are avoided with `dragRef`, `allNodesRef`, and `dataRef` — always update these refs when state changes.

### Rendering

- Tree nodes render as `<div>` absolutely positioned over an `<svg>` (edges drawn as `<path>` with L-shaped connectors).
- The matrix below is a plain `<table>` with leaf nodes as columns and test cases as rows.
- All styling is inline CSS with a dark theme (`#0A1118` background).

## Conventions

- Plain JS/JSX (no TypeScript).
- All IDs generated by `uid()` — a module-level auto-incrementing counter.
- ESLint rule: unused vars allowed if name matches `^[A-Z_]`.
